## Data And Algorithm Assessment

- Parameter iteration matches the legacy data order. Python builds `list(itertools.product(thresholds, min_lengths_ms))` with thresholds `[-45, -40, -35, -30, -25, -20]` and min lengths `[8000, 6000, 4000, 2500, 1500]` (`inference/API/slicer_api.py:323`, `inference/API/slicer_api.py:326`). Rust uses the same threshold-outer/min-length-inner `flat_map` order in `GridSearchConfig::parameter_grid` (`rewrite-in-rust/rust/crates/v2m-core/src/slicer_grid.rs:42`, `rewrite-in-rust/rust/crates/v2m-core/src/slicer_grid.rs:46`). Fixture line 1 expands and checks all 30 call pairs (`rewrite-in-rust/fixtures/slicer_grid_search_policy_core.jsonl:1`).
- The scoring formula is output-equivalent for the covered numeric domain. Python computes per-chunk duration with `len(c["waveform"]) / sr`, adds `1.5x` short penalties, `1.0x` long penalties, and the chunk-count deviation penalty `abs(len(chunks) - len(waveform) / sr / average_target) * 0.5` (`inference/API/slicer_api.py:346`, `inference/API/slicer_api.py:352`, `inference/API/slicer_api.py:360`). Rust computes the same terms in `score_chunks` (`rewrite-in-rust/rust/crates/v2m-core/src/slicer_grid.rs:171`, `rewrite-in-rust/rust/crates/v2m-core/src/slicer_grid.rs:182`, `rewrite-in-rust/rust/crates/v2m-core/src/slicer_grid.rs:186`). Fixture line 3 covers long, short/count selection pressure and verifies the expected winner (`rewrite-in-rust/fixtures/slicer_grid_search_policy_core.jsonl:3`).
- Stereo `len()` behavior is intentionally preserved. Python's grid scorer uses `len()` rather than `shape[-1]`, so a channel-major stereo array is scored by channel count, not sample count (`inference/API/slicer_api.py:346`, `inference/API/slicer_api.py:360`). Rust uses `Waveform::outer_len()`, which returns sample count for mono and channel count for stereo (`rewrite-in-rust/rust/crates/v2m-core/src/slicer_grid.rs:172`, `rewrite-in-rust/rust/crates/v2m-core/src/slicer_grid.rs:186`, `rewrite-in-rust/rust/crates/v2m-core/src/slicer_segment.rs:24`). Fixture line 5 pins that legacy stereo scoring quirk (`rewrite-in-rust/fixtures/slicer_grid_search_policy_core.jsonl:5`).
- Tie behavior is strict-first, matching Python. Python updates only when `score < best_score`, so equal-score candidates retain the earlier candidate (`inference/API/slicer_api.py:365`). Rust uses the same strict comparison and only replaces `best_chunks` inside that branch (`rewrite-in-rust/rust/crates/v2m-core/src/slicer_grid.rs:139`). Fixture line 2 proves the first equal-scoring candidate wins (`rewrite-in-rust/fixtures/slicer_grid_search_policy_core.jsonl:2`).
- Error and empty-output handling are compatible with the algorithm boundary. Python wraps constructor, slice, and scoring in one `try` and `continue`s on exceptions, while empty chunk lists are skipped before scoring (`inference/API/slicer_api.py:333`, `inference/API/slicer_api.py:343`, `inference/API/slicer_api.py:370`). Rust skips closure errors, empty chunks, zero-rate/zero-average-target scoring failures, and returns an empty vector if no candidate wins (`rewrite-in-rust/rust/crates/v2m-core/src/slicer_grid.rs:127`, `rewrite-in-rust/rust/crates/v2m-core/src/slicer_grid.rs:131`, `rewrite-in-rust/rust/crates/v2m-core/src/slicer_grid.rs:135`, `rewrite-in-rust/rust/crates/v2m-core/src/slicer_grid.rs:148`).
